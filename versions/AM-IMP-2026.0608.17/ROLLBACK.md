# Rollback

If the task hierarchy architecture causes problems:

1. Stop writing new values to `母任務` and `子任務`.
2. Continue using flat task lists and the `總控專案` relation temporarily.
3. Do not delete existing task pages, child task links, source conversations, attachments, or dossier body content until project-local records have been reviewed.
4. Remove or adjust task dossier body sections only from pages where they were incorrectly applied.
5. Update the project manifest to `Blocked` if the project cannot safely migrate yet.

Rollback must not delete source conversations, attachments, project records, or task records.
