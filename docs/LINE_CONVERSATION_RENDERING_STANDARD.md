# User UI LINE 對話引用呈現標準

本文件定義 User UI 只要引用 LINE 對話內容時，必須如何呈現。

核心原則：User UI 不重新設計 LINE 對話內容的樣式。只要畫面上要呈現來自 LINE 對話的內容，就必須回到專案本地資料庫中的 `LINE 對話主檔` 或 `LINE 對話組檔`，並依照該檔案內文的樣式呈現。

參考樣式來源：

- SevenAM 範例：`Andy & Seven`
- 來源資料庫：`Seven LINE 對話主檔`
- 範例頁：`https://app.notion.com/p/37951c686dac815caa80f59631e4d06d`

AMCore 只保存呈現規範，不保存該頁的 live 對話內容、圖片檔或專案資料。

## 適用範圍

下列 User UI 位置只要引用 LINE 對話內容，都必須使用本標準：

- 任務內頁的來源證據。
- 任務內頁的狀態變更證據。
- 任務內頁的圖片、照片或附件證據。
- LINE 群組與訊息頁。
- 對話搜尋結果。
- 報告頁或報告紀錄中引用的 LINE 對話片段。
- 任務校準頁中引用的 LINE 對話片段。

若內容來源不是 LINE，例如會議記錄、報告紀錄或手動任務，則不使用本標準。

## 資料來源

User UI 需要呈現 LINE 對話時，必須優先讀取專案本地的 LINE 對話主檔或 LINE 對話組檔。

必須使用的來源資訊：

- 日期時間。
- 群組名稱或對話名稱。
- 發訊者名稱。
- 訊息類型，例如文字訊息、圖片、檔案、貼圖或其他訊息。
- 訊息內容。
- 圖片、照片、檔案、貼圖或附件連結。
- 對話主檔頁面連結。

不得用任務資料庫裡的簡化文字取代完整 LINE 對話主檔格式。任務資料庫可以保存摘要與引用關係，但正式顯示時應回查 LINE 對話主檔。

## 標題列格式

每一則 LINE 訊息引用都必須先顯示一行中繼資訊標題列。

標準格式：

```text
【YYYY/M/D 上午/下午H:MM】群組名稱 - 發訊者名稱（訊息類型）
```

若來源資料使用 24 小時制，也可以保留來源檔原格式：

```text
【YYYY/MM/DD HH:MM】發訊者名稱：訊息類型
```

User UI 不應自行混用第三種格式。若同一段引用來自同一個 LINE 對話主檔，應完全沿用該主檔內文的時間、名稱、標點、空格與訊息類型寫法。

## 顏色規則

User UI 應沿用 LINE 對話主檔中的標題列顏色。

範例中可見的標準：

- 一般 LINE 來源訊息標題列使用藍色。
- Seven Jr. 或系統助理發出的訊息標題列使用橘色。
- 內容文字使用一般內文字色，不另加醒目背景。

在 HTML User UI 中，可用等價 class 呈現：

```html
<div class="line-message-title line-message-title--source">【2026/6/8 下午11:44】Andy & Seven - Andy Tsai（圖片）</div>
<div class="line-message-body">附近有一間便利超商距離五分鐘</div>

<div class="line-message-title line-message-title--assistant">【2026/06/10 00:08】Seven Jr.：文字訊息</div>
<div class="line-message-body">Andy，想跟你確認一下...</div>
```

`line-message-title--source` 必須視覺對應 Notion 內文的藍色標題。

`line-message-title--assistant` 必須視覺對應 Notion 內文的橘色標題。

## 內容文字格式

內容文字必須直接接在標題列下方。

規則：

- 保留原始換行。
- 保留原始 URL。
- 保留原始文字、標點、emoji 與貼圖標記。
- 不把對話內容改寫成摘要。
- 不把每則訊息包成另一套任務卡片樣式。
- 不在 LINE 引用區塊中加入 AM 自己的判斷文字。

AM 的判斷、任務狀態、下一步與摘要應放在任務頁其他區塊，不得混入 LINE 原始對話引用區。

## 圖片與多媒體格式

圖片訊息必須在標題列下方直接顯示圖片。

標準格式：

```text
【YYYY/M/D 上午/下午H:MM】群組名稱 - 發訊者名稱（圖片）
![圖片名稱或 message id](圖片 URL 或本地預覽檔)
```

規則：

- 圖片必須直接帶上，不得只顯示 `圖片：617...`、`[image]` 或 message id。
- 圖片可使用專案本地預覽檔、專案本地附件檔，或仍有效的外部圖片 URL。
- 圖片應可點擊開啟原圖或附件紀錄。
- 如果圖片暫時無法取得，仍必須顯示 placeholder，並包含 message id、時間、群組、發訊者與可用附件連結。
- 檔案訊息應在標題列下方顯示檔名與附件資料庫連結。
- 貼圖訊息應保留 `[sticker] package:... sticker:...`，並在可用時附上 LINE sticker shop page。

## 區塊樣式

LINE 對話引用區塊的視覺目標是「像 LINE 對話主檔內文」，不是「像任務卡片」。

因此：

- 不使用任務頁專屬的證據卡片標題樣式取代 LINE 訊息標題列。
- 不使用不同頁面各自定義的對話框、泡泡、時間線或摘要卡片。
- 不把圖片另外搬到獨立的圖片卡片後才呈現；圖片應留在對應訊息標題列下方。
- 任務頁、LINE 群組頁、搜尋結果、報告引用與校準引用都必須重用同一個 LINE conversation renderer。

## 與任務頁的關係

任務頁可以有自己的任務控制區、目前判斷、下一步與稽核區。

但只要任務頁進入「來源 LINE 對話」或「LINE 證據」段落，該段落必須使用本標準。

任務頁不得用另一套「證據卡片」重新包裝 LINE 訊息。若需要說明某則訊息造成什麼任務狀態變更，應在 LINE 引用段落外另設 AM 判斷文字，並連回該訊息。

## 驗證清單

每次重產 User UI 時，必須檢查：

- 所有 LINE 對話引用都能追溯到專案本地 LINE 對話主檔或 LINE 對話組檔。
- 每則引用都有日期時間、群組名稱或對話名稱、發訊者名稱與訊息類型。
- 標題列顏色與 LINE 對話主檔一致。
- 內容文字直接接在標題列下方，並保留換行。
- 圖片訊息直接顯示圖片，不只顯示 message id。
- 檔案訊息顯示檔名與附件連結。
- 貼圖訊息保留貼圖資訊與可用連結。
- 任務頁、LINE 群組頁、搜尋結果與報告引用使用同一套 renderer。
- 沒有其他舊版對話卡片、泡泡或時間線樣式混用。
