# AM 任務判斷規則載入標準

本文件定義 AM 在進行任務擷取、任務狀態判斷、任務進度更新、任務合併或不建任務判斷前，必須如何載入最新任務判斷規則。

這是 AM 的核心規定。每小時 LINE 檢查、會議任務同步、報告寫回、任務校準重判，以及任何自動建立、更新、合併、封存或完成任務的流程，都不得只依賴程式內建的固定判斷。

## 核心原則

每次判斷任務前，系統必須先載入最新規則。

必須載入的規則來源：

1. AMCore 共用任務判斷規則。
2. 專案本地任務判斷規則資料庫。
3. 專案本地校準案例整理出的學習規則。
4. 使用者從 User UI 手動新增且已啟用的規則。

這些規則載入後，才可以開始判斷新 LINE 訊息是既有任務的更新，還是新的事件任務。

## 專案本地規則庫位置

各專案的 live 規則必須存在各專案自己的資料庫，不得混入 AMCore。

| 專案 | 規則庫環境變數 |
| --- | --- |
| HOZO AM | `HOZO_JUDGMENT_RULES_DATA_SOURCE_ID` |
| SevenAM | `SEVEN_JUDGMENT_RULES_DATA_SOURCE_ID` |
| 共用命名 | `PROJECT_JUDGMENT_RULES_DATA_SOURCE_ID` |

AMCore 只保存共用標準、模板、schema、範例與升級包；不保存專案 live 任務資料、LINE 訊息、客戶資料或專案本地規則內容。

## 適用流程

本標準適用於：

- 每小時 LINE 任務核對。
- LINE 訊息是否形成任務的判斷。
- 新訊息是既有任務更新或新事件的判斷。
- 任務狀態、進度、負責人、下一步或期限更新。
- 會議 checkbox 任務同步與會議內容更新。
- 各時段報告候選項目判斷與寫回。
- 任務校準後的重判。
- 自動建立、合併、封存、完成或不建立任務的任何流程。

## 載入順序

建議載入順序：

1. 載入 AMCore 共用規則。
2. 載入目前專案的專案專屬規則。
3. 載入目前專案的校準學習規則。
4. 載入目前專案由 User UI 手動新增的啟用規則。

若專案規則與 AMCore 共用規則衝突，應優先套用更具體且已啟用的專案規則，並記錄該次判斷使用的規則來源。

## 規則篩選

載入規則後，判斷流程應依下列條件篩選可用規則：

- `Status`：只使用啟用、已驗證、已安裝或依專案政策可用的規則。
- `Applies To`：規則必須適用於目前專案、目前流程或 AMCore 共用範圍。
- `Trigger Pattern`：規則必須符合目前訊息、對話串、任務、會議、報告或來源情境。
- `Exceptions`：若例外條件排除目前情境，該規則不得套用。
- `Checklist Placement`：若規則指定只用於特定檢查點，必須在正確檢查點才套用。

## 規則使用範圍

載入後的規則必須被納入以下判斷：

- 是否擷取任務。
- 是否更新既有任務，而不是建立新任務。
- 是否合併、壓制或封存重複任務。
- 任務狀態是否改變。
- 任務進度是否有更新。
- 任務應歸屬哪個專案或專案目標。
- 任務負責人或責任方向是否改變。
- 任務下一步應寫入什麼欄位內容。
- 是否屬於背景訊息、系統操作、測試訊息、純知識分享或不建立任務的情境。
- 是否需要專案負責人確認後才能完成或關閉。

## 稽核要求

當某條規則實質影響判斷結果時，系統應在任務證據、訊息判斷紀錄或校準 trace 中記錄：

- 套用的規則名稱或 ID。
- 規則來源。
- 規則造成的判斷效果。
- 判斷時間。
- 對應的來源訊息、會議紀錄或報告。

這樣日後回看任務時，才能知道任務為什麼被建立、更新、合併、暫停、完成或判定不建立。

## 載入失敗處理

規則載入失敗時，不得靜默忽略。

系統必須：

1. 在該次 hourly run、報告 run 或任務判斷 log 中記錄 warning。
2. 在可見報告或系統狀態中留下可追蹤線索。
3. 使用最安全的 fallback 判斷。
4. 在規則恢復載入後，允許對受影響訊息或任務重新判斷。

涉及金錢、法律、稅務、HR、合約、客戶承諾、部署或外部承諾的任務，在規則載入失敗時不得自動完成或對外行動。

## 驗證清單

每個專案導入本標準後，至少要能驗證：

- 專案本地規則庫環境變數存在。
- 每小時 LINE 任務判斷在讀取訊息前會先載入規則。
- User UI 手動新增的規則會進入專案本地規則庫。
- 載入後的規則會依狀態、適用範圍、觸發條件與例外條件篩選。
- 任務建立、更新、合併、狀態變更或不建立任務時，可以看到使用的規則線索。
- 規則載入失敗時會留下 warning，而不是靜默略過。
